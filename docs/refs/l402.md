# L402 reference notes

L402 is useful precedent for combining HTTP 402, an invoice, a bearer capability, and an authenticated retry.

Fiber Paid HTTP applies these lessons:

- bind the capability to challenge, resource, amount, method, payment hash, issuer, and expiry;
- treat the capability and preimage as secrets;
- enforce atomic single-use redemption;
- keep network-specific settlement behind the Fiber method adapter;
- map the entrance into the same MPP-draft credential verifier.

The implemented token is project scoped:

```text
fiber-l402-capability-v1.<jcs-base64url>.<hmac-sha256-hex>
```

The retry is `Authorization: L402 <capability>:<preimage>`. No third-party byte-format compatibility is claimed.
