# L402 Reference Notes

L402 combines HTTP 402, Lightning invoices, and bearer credentials. The important precedent is not Lightning itself, but the split between payment request, proof/credential, and authenticated retry.

Fiber Paid HTTP design lessons:

- Bind every credential to the challenge, resource, amount, method, and expiry.
- Treat credentials as bearer secrets and enforce single-use redemption.
- Return a receipt after successful verification.
- Keep payment-network-specific details behind method adapters.

Current F-L402 implementation:

- Fiber Paid HTTP does not use Lightning invoices.
- Fiber Paid HTTP implements an application-level `fl402-macaroon-v1` token for Fiber invoices.
- The token is HMAC-signed canonical JSON with first-party caveats, not a Lightning Labs macaroon byte format.
- `Authorization: L402 <macaroon>:<preimage>` is converted into the internal Fiber Paid HTTP credential path, so replay storage and receipts remain centralized.
