# Contributing

Fiber Paid HTTP is a Rust-first paid HTTP gateway and protocol toolkit. Rust is the trusted production verifier; TypeScript is SDK, middleware, compatibility, evidence-console, and vector tooling.

## Setup

Use Node 24 and pnpm 10.12.1 or newer:

```bash
pnpm install --frozen-lockfile
pnpm build
```

Rust uses the workspace toolchain from `Cargo.toml`:

```bash
cargo fmt --all --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
```

## Checks Before A PR

Run the fastest relevant checks for your change:

```bash
pnpm lint
pnpm typecheck
pnpm test
cargo test --workspace
```

For protocol, verifier, Fiber RPC, storage, gateway, or evidence changes, also run:

```bash
pnpm exec fiber-paid-http vectors verify
cargo run -p fiber-paid-http-cli -- vectors verify
bash scripts/fiber_paid_http_canonical_gate.sh
```

Run the full gate before release or hackathon submission:

```bash
bash scripts/fiber_paid_http_gate.sh
bash scripts/fiber_paid_http_rust_gate.sh
bash scripts/fiber_paid_http_canonical_gate.sh
```

## Adding A Vector

Vector changes must keep TypeScript and Rust in lockstep.

1. Add or regenerate the fixture under `test-vectors/`.
2. Update `packages/cli/src/vectors.ts` in the `verifyVectorInput` path.
3. Update `crates/fiber-paid-http-core/src/lib.rs` in the `verify_vector_input` path.
4. Run both vector verifiers.
5. Confirm `reports/canonical-core-parity.json` has no mismatches.

Use the current terminology: 16 deterministic conformance vectors plus 2 live-Fiber evidence fixtures, tracked as 18 shared vector files.

## Fiber Evidence

`pnpm test:fiber` always runs the visible preflight. The live payment test only runs when `RUN_FIBER_E2E=1`, `FIBER_MODE=local|testnet`, payer/payee RPC URLs, and a signing secret are configured.

Local:

```bash
scripts/fiber_local_network.sh up
RUN_FIBER_E2E=1 \
FIBER_MODE=local \
FIBER_PAYEE_RPC_URL=http://127.0.0.1:21716 \
FIBER_PAYER_RPC_URL=http://127.0.0.1:21714 \
FIBER_PAID_HTTP_SECRET="$(openssl rand -hex 32)" \
pnpm test:fiber
```

Testnet evidence requires funded payer/payee Fiber nodes with peers and `ChannelReady` channels. See `docs/fiber-testnet-e2e.md`.

## Documentation Rules

- Do not use maintainer-specific absolute paths in user-facing docs. Prefer repo-root commands, `$FIBER_REPO`, `$HOME`, or `/path/to/...` placeholders.
- State whether a flow is local, testnet, production-like, historical evidence, or simulated.
- Do not treat TypeScript as the trusted verifier boundary. Rust owns canonical verification.
- Keep examples honest. If an example is README-only, say so at the top.

## Pull Request Scope

Keep PRs narrow. In a dirty worktree, stage only files that belong to the change. Do not rewrite historical reports unless the change is explicitly an archive/status banner or the report is being regenerated by a documented gate.
